## Import

Imports use the Terraform resource `id`, which is built from `user_id`, `authority_name`, and `external_key`. Each segment is URL path–escaped and joined with `/`, so characters such as `/` in a URL `authority_name` (or in `external_key`) appear as percent-encoded sequences in `id` rather than as extra delimiters. Use the `id` value from state or export output as the import identifier (for example `terraform import genesyscloud_externalusers_identity.example "<id>"`). Existing state whose `id` was formed without escaping could not be parsed correctly if those fields contained `/`; after upgrading the provider, run `terraform apply` so `id` is refreshed to the encoded form.
